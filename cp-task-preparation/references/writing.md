# Writing Conventions

Use this reference when writing or cleaning statements, editorials, proofs, and mathematical documentation.

## Markdown Math

- Prefer LaTeX math `$...$` for variables, expressions, constraints, recurrences, complexity, proof notation, and formula derivations.
- Use code formatting for filenames, function names, code symbols in prose, shell commands, literal input/output syntax, and non-mathematical technical text.
- In statements and editorials, write "array $A$ of length $N$" and "$1 \le N,Q \le 10^5$" instead of using backticks for mathematical notation.
- Keep full signatures, input formats, and shell commands in fenced code blocks.
